BeanBase

The one purpose of creating BeanBase is to make RedBeanPHP: Powerful ORM even easier to use...Skip to wiki (API docs and manual)

A slight taste with a use case:

The developer should be able to programmatically store a job entry into the database, given the job information (title, description), without touching a line of SQL.

With RBB:

$data['title'] = $_POST['title'];
$data['description'] = $_POST['description'];

$bean = RBB::create( 'job', $data );

$bean_id = R::store( $bean ); // Use vanilla RedBeanPHP to store the bean

Now with BaseModel:

class JobModel extends BaseModel {
  public $type = "job";
}

$data['title'] = $_POST['title'];
$data['description'] = $_POST['description'];

$job = new JobModel();
$bean = $job->post( $data );
